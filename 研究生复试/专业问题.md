**2.操作系统：**

```
- 中断相关概念；内中断和外中断的类型；优先级；外中断的流程；中断隐指令；

	中断分为外中断和内中断，内中断也称异常，分为故障、内陷与终止。
	一般说的中断都是外中断，在每条指令周期的最后会自动检测有没有(外)中断发出(有一条自带优先级的队列),有则进行中断处理，而异常可以在指令执行当中，比如缺页异常。
	外中断由程序主动发出，基本的用处大概起源于进程切换等。当某进程被阻塞，通过中断通知进行进程切换。
	异常的优先级最高，大于外中断。


- 操作系统的启动流程
	首先激活CPU -> (激活中断向量表) -> 启动BIOS进行系统自检 -> BIOS在CMOS中找到BOOT Squence中找到主引导记录MBR，也称引导扇区。
	引导扇区记录着：
	①操作系统本体所在位置，CPU记录该位置
	②引导扇区中存在磁盘分区表，CPU根据分区表扫描每个分区，找到其中的活动分区，活动分区开头记录着分区引导记录PBR
	CPU根据PBR找到操作系统引导程序所在，并执行该程序引导操作系统。


- 进程基本概念/组成/性质；进程的状态转移；进程的三种通信；管道的特性、本质
	进程是程序执行的基本单位，由程序段、数据段与进程控制块(PCB)组成。进程的最基本特性是动态性。
	进程间通信：共享文件、消息传递、管道
	管道本质是一种特殊的文件，是互斥的，读写特点，子进程可以继承父进程的管道 => 多个写一个读。
	

- 线程的概念；线程的三种实现方式与特点；多线程模型
	进程仍是分配资源的基本单位，但不再是调度的基本单位。线程控制块
	用户级线程：纯粹的应用程序，由在用户态实现的线程库操作实现。本质属于一个进程，因此被阻塞时全部被阻塞
	内核级线程：不会全部被阻塞，内核态中存在多个，可以进行统一进程内的线程调度。缺点是慢，因为所有的调度都要通过访管进入内核。
	混合...


- 进程调度(6种)
	先来先服务、短作业优先、优先级、时间片、高响应比队列、多级反馈队列

- 记录型信号量；管程与条件变量
	signed：发出信号，释放一个资源
	wait：申请一个资源
	管程：为避免每个进程之间都拥有自己的信号量，用户编写程序时还需要与对方商议等。操作系统自己实现一个统一的信号量分配机构。条件变量是基于信号量之上的，因为不同的变量之间可能有不同的互斥方式，为每一种互斥方式声明一个名字——条件变量。
	条件变量只记载有没有，不记载互斥的数量。


- 死锁的定义；死锁产生的四个必要条件、各条件的含义
	互斥条件(独占)：同一资源同一时刻内能且仅能被一个进程独占使用。
	不可剥夺：不存在一个进程抢占另一个进程的资源。
	请求并保持：至少拥有一个已经请求的，并不释放该请求的。
	循环等待：成链。
	银行家算法大致流程，各列的含义


- 段、页存储的基本思想

- 三种I/O控制方式及各自特点
	CPU循环检查
	中断打断
	DMA

- 虚拟存储的简介及其好处
```

**3.计算机组成原理：**

```
- 总线的定义与性质
	总线是为各部件提供分时共享的信息传输通道。基本性质是分时与共享
	分时：同一条总线同一时间内只能为同一组组件使用。
	共享：可以一个发，多个收


- 中断的流程
	关中断 -> 保存断点(PC、PSW) -> 查中断向量表(中断服务程序寻址)
    上述由中断隐指令(硬件)完成
    保存现场 -> 开中断 -> 中断服务程序
    开中断的目的是为了实现中断嵌套
    关中断 -> 恢复现场 -> 开中断


- DMA的定义、基本性质、访存方式
	是外部控制器与内存的直接数据通道，只需要CPU进行开启和收尾工作，最中间不需要任何CPU操作。
	但对CPU也有影响。即争用内存
	直接暂停CPU的访存
	周期挪用(抢劫。I/O访存优先级更高，读取到的数据来不及存储就可能丢失)
	交替存取。if CPU的工作周期很长，可以趁CPU不访存时访存


- DMA与中断响应的区别
	DMA无需保护现场等操作，不发生切换行为
	DMA请求可以在一个机器周期的任何阶段被接受；
	注意这里的接受是DMA请求。实际上是，DMA可以在任何时间被CPU授权开始。
	而CPU收尾DMA的操作实际上还是通过中断进行的，因此只能在机器周期的末尾。
```

**4.计算机网络**：

```
- 计算机网络的划分(三种)
	电路交换网络(打电话)、报文交换网络(写信)、分组交换网络；分组交换网络就是在报文的基础上加上标准化规范，让不同网络不同发送者能够使用相同的接口发送不同的内容。

- OSI参考模型(7层)简介即基本功能及特殊协议
	物理层 -> 数据链路层 -> 网络层 -> 传输层 -> 会话层-> 表示层 -> 应用层；
	会话层：进程之间相互通信的一层
	表示层：数据通信的编码等，将信息转为二进制内容
	
- TCP/IP模型(4层)、与OSI的对比
	网络接口层 -> 网络层 -> 传输层 -> 应用层
	TCP/IP认为可靠传输是传输层负责实现，因此网络层提供无连接，运输层负责有/无连接
	OSI网络层提供有/无连接，运输层只提供有连接。
	

--------------------------物理层--------------------------
- 奈奎斯特定律、香农定律
	二者都是计算最大信息传送速率的方法
	奈奎斯特定理：用带宽规定；
	香农定律：用链路中的噪声判断，计算信噪比

- 编码与调制的概念、数字转数字的编码
	编码：XX -> 模拟信号；调频调幅调相
	编码 XX -> 数字
	数字转数字的编码：
	归零：高1低0，但每次中间必降落
	非归零：高1低0
	反向非归零：变是0不变是1
	曼彻斯特：看前半部分
	差分曼彻斯特：间隔不变是0变是1

- 数据报与虚电路
	网络层、虚电路是虚的，建立一条协商好的通道，则以后每次发送时无需再知道目的地址，只需了解所创建的虚电路号码。(TCP、视频聊天、网络电话)

- 工作在物理层的设备(中继器、集线器)
	既不划分冲突与也不划分广播域。单纯信号增强放大作用

--------------------------数据链路层--------------------------
- 数据链路层基本
	透明传输、封装成帧

- 流量控制、可靠传输与滑动窗口机制(本质，什么是流量控制)
	可靠传输：选择重传与确认机制
	流量控制：因接收方接受速率是有限的，而一旦超出其能力，多余的数据会丢失，因此接收方对发送方做出发送速率的限制。
	后退N帧协议：发送窗口N，接收窗口1，累计确认。错误全部重传
	选择重传协议：N-N；使用错误帧

- 介质访问控制(介质划分、随机访问、轮询)(原理、来源)
	介质访问控制是在局域网等中：当两个信源同时发送数据会造成冲突的网络上的，为解决碰撞无休止的一直发生而产生的
	介质划分：信道划分复用技术，频分时分空分复用，码分多址(将不同频率的信号合成为一条，实际上不是无限的，对信道有要求)
	随机访问：本质是随机逼退时间
	CSMA/CD：载波监听多路访问()碰撞检测：用于有线网。
	当遇到碰撞时，立刻停止，而不会发送信号通知发起站，显然是因为通知这个行为同样会碰撞，因此，有最短帧长的概念。
	最短帧长：是这样的，默认发送第一个bit需要0s，默认最极限情况第一帧已经到了目标，发生碰撞，即一个来回的传播时间内，发出停止了、发送站不再监听碰撞，因此发送站不会知道发生了碰撞，因此不足最短帧长的需要补足。
	CSMA/CA(RTS、CTS)：碰撞避免，确定帧间间隔

- 局域网、以太网、IEEE
	MAC帧，数据链路层。

- 广域网
	PPP协议(点对点)：广域网中使用，是使用拨号或专线方式连接不同路由器、转发器、网桥的一种方式。其串行、无错误检测、纯软件实现、全双工、允许不同网络层协议、
	HDLC协议

- 数据链路层设备(网桥、交换机)

- 冲突域与广播域

--------------------------网络层--------------------------
- 网络层的主要任务
	利用IP协议。连接不同异构网络，提供统一的虚拟接口。

- SDN(软件定义网路)
	由服务器集中统一路由器转发。

- 路由算法(链路状态、距离向量)
	决定IP分组下一跳去往哪个分组的算法，目的当然是最快最安全的到达终点。
	RIP路由信息协议：每次只将自己的全部路由表传递给自己的相邻路由，每个路由器只知道自己所有临近路由器的信息。路由器网络需要收敛。
	基于UDP的应用层协议
	OSPF开放最短路径优先协议：根据dijkstra，一次性算出(所有路由器全知道)真个网络的完整拓扑结构。使用泛洪法，依次传递。
	
- 域间路由选择(外部网关协议)
	BGP协议：使用TCP在自洽系统之间一次性大量快速安全传递信息。

- IP协议基础、子网划分、子网掩码、无分类编址CIDR	

- 地址解析协议ARP
	在局域网内部，两个路由器之间，IP报文只有下一跳的IP地址，要使用地址转换协议将其转换为MAC地址来指导路中间的交换器与网桥。

- 网络地址转换NAT
	专用网、私有网的专门设备。工作在传输层(有端口)，每个设备都有一个自己的IP过于浪费，因此采用该设备，专门设置内网虚拟IP，该设备就是转换装置。


- 动态主机配置协议DHCP
	没有自己的IP地址的设备，想远程服务器申请。全部采用广播。

- 网际报文控制协议ICMP
	纠错与检错专用格式的报文。

- IP组播与IGMP因特网组管理协议


--------------------------传输层--------------------------
- UDP与TCP、三次握手与四次挥手的流程与原理
	UDP：无连接，不可靠的
	TCP：有连接、可靠的
	三次握手：确认了双方的状态全部正常，避免以往超时的SYN影响(非三次握手期间期待的序号都将被抛弃)。
	四次挥手
	
- TCP拥塞控制(慢开始+拥塞避免、快重传+快恢复)
	快重传：快速发送三次冗余ACK，会跳过乘法增加环节。

--------------------------应用层--------------------------
- 域名解析协议DNS、
	递归查询
	递归与迭代相结合
	
- 文件传输协议FTP

- 简单邮件传输协议SMTP、POP3邮局协议
```
